## 5 栈与队列

### 5.1 栈
- 基于链表的实现
- 基于数组的实现

时间效率：
- 数组，触发扩容机制，效率下降
- 链表，可以提供更加稳定的效率表现

空间效率：
- 数组，倍率扩容，一定的空间浪费
- 链表，额外空间存储指针，节点空间占用大

典型应用：
- 浏览器中的后退与前进
- 软件中的撤销与反撤销
- 程序内存管理（函数调用、递归等）

### 5.2 队列
先进先出，头删尾插

典型应用：
- 淘宝订单
- 待办事项

### 5.3 双向队列
- 队首队尾都可入队出队
- 双向队列兼具栈与队列的逻辑，因此它可以实现这两者的所有应用场景，同时提供更高的自由度

典型应用：
- 软件的撤销和反撤销

撤销通常会有步数的限制，如果栈满，则需要在栈底进行出队操作，这个时候就需要双向队列来实现，但是撤销的逻辑仍然遵循栈的先进后出原则

## 6 哈希表

在哈希表中进行增删查改的时间复杂度都是 $O(1)$

|          | 数组   | 链表   | 哈希表 |
| -------- | ------ | ------ | ------ |
| 查找元素 | $O(n)$ | $O(n)$ | $O(1)$ |
| 添加元素 | $O(1)$ | $O(1)$ | $O(1)$ |
| 删除元素 | $O(n)$ | $O(n)$ | $O(1)$ |


哈希 => 哈希冲突 => 扩容 => 效率低

哈希表结构改良方法：
- 链式地址
- 开放寻址
  - 线性探测
  - 平方探测
  - 多次哈希
